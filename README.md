# Алгоритмы защиты эхо сигналов от несинхронных импульсных помех

Комплексная система для исследования и сравнения алгоритмов защиты эхо сигналов от несинхронных импульсных помех. Дипломная работа включает теоретическое обоснование, реализацию на C++ и экспериментальное сравнение алгоритмов фильтрации с адаптивным выбором оптимального метода.

## 🚀 Ключевые особенности

- **6 алгоритмов фильтрации** включая робастный фильтр Винера для импульсных помех
- **Автоматическая классификация сигналов** и адаптивный выбор оптимального фильтра (`-f auto`)
- **Комплексный генератор тестовых данных** с 5 типами эхо сигналов и 4 типами помех
- **Автоматическая система тестирования** с детальным анализом производительности
- **Подробная документация** с теоретическим обоснованием и практическими рекомендациями
- **Гибкая архитектура** для легкого добавления новых алгоритмов

## 🤖 Адаптивная система классификации и выбора фильтра

### Обзор

В проекте реализована **автоматическая система**, которая анализирует поступающий сигнал, определяет его тип и применяет наиболее подходящий алгоритм фильтрации без участия пользователя. Система состоит из двух компонентов:

- **`SignalClassifier`** ([`src/signal_classifier.h`](src/signal_classifier.h)) — извлекает набор статистических признаков и по правилам классифицирует тип сигнала
- **`AdaptiveFilterSelector`** ([`src/adaptive_filter_selector.h`](src/adaptive_filter_selector.h)) — по результату классификации создаёт и запускает оптимальный фильтр

---

### Алгоритм классификации сигналов

#### Шаг 1 — Извлечение признаков (Feature Extraction)

Для каждого входного сигнала `x[n]` длиной `N` вычисляется вектор из 7 числовых признаков.
Все признаки вычисляются только во временно́й области — FFT не используется.

---

**1. Куртозис (Kurtosis)**

```
κ = (1/N · Σ(x[n] − μ)⁴) / σ⁴
```

Мера «остроты» распределения значений сигнала.

| Тип сигнала | Типичное κ |
|-------------|-----------|
| Синус | ~1.5 |
| Меандр | ~1.0 (бимодальное) |
| Треугольник | ~1.8 |
| Гауссов шум | ~3.0 |
| Импульсные помехи | **>> 10** |

---

**2. Пик-фактор (Crest Factor)**

```
CF = max|x[n]| / RMS(x)
```

Отношение пикового значения к среднеквадратическому. Высокий CF указывает на наличие резких пиков относительно общего уровня сигнала.

| Тип сигнала | CF |
|-------------|---|
| Синус | √2 ≈ 1.41 |
| Меандр | 1.0 |
| Эхо-импульс (с тишиной) | **>> 3** |

---

**3. Частота пересечений нуля (Zero-Crossing Rate)**

```
ZCR = (число смен знака x[n]) / (N − 1)
```

Характеризует «скорость колебаний» сигнала. Для периодических сигналов ZCR ≈ 2·fₙ (двойная нормированная частота). Для эхо-сигнала, большинство отсчётов которого равны нулю — ZCR близко к нулю.

---

**4. Разреженность (Sparsity)**

```
sparsity = #{|x[n]| < ε · max|x|} / N,    ε = 0.05
```

Доля «почти нулевых» отсчётов. Для эхо-радиолокационных сигналов (прямоугольный, гауссовский импульс + тишина) значение превышает 0.65–0.85.

---

**5. Гладкость (Smoothness)**

```
smoothness = (1/(N−1) · Σ|x[n] − x[n−1]|) / RMS(x)
```

Среднее абсолютное первое различие, нормированное на RMS.

| Тип сигнала | Поведение |
|-------------|-----------|
| Синус | Малое (медленные изменения) |
| Меандр | Умеренное (редкие большие скачки) |
| Белый шум | Большое |

---

**6. Отношение локальных дисперсий (LocalVarRatio)**

```
LVR = max(Var_local[k]) / mean(Var_local[k])
```

Сигнал разбивается на непересекающиеся окна длиной `localWindow` (по умолчанию 31 отсчёт). В каждом окне вычисляется локальная дисперсия. LVR = отношение максимальной к средней.

- **Стационарный синус** → LVR ≈ 1 (дисперсия равномерна по всему сигналу)
- **Сигнал с импульсными помехами** → LVR >> 1 (окно с выбросом имеет многократно большую дисперсию)

---

**7. Нормированный размах (RangeRatio)**

```
RangeRatio = (max(x) − min(x)) / 2 / RMS(x)
```

- Синус: ≈ 1.41 (амплитуда / RMS = √2)
- Меандр: ≈ 1.0
- Сигнал с огромными выбросами: **>> 3**

---

#### Шаг 2 — Классификация по правилам (Rule-Based Classifier)

Решающие правила применяются **последовательно** (первое сработавшее — итоговый класс):

```
IF  κ > 10  AND  RangeRatio > 3
    → NOISY  (несинхронные импульсные помехи)

ELSE IF  sparsity > 0.65  AND  CF > 3
    → ECHO   (разреженный импульсный сигнал)

ELSE IF  κ < 2.5  AND  smoothness > 0.05  AND  RangeRatio < 2
    → SQUARE (меандр)

ELSE IF  κ < 3.0  AND  0.01 < smoothness < 0.15  AND  ZCR > 0.05
    → TRIANGLE (треугольный/пилообразный)

ELSE IF  κ < 4.0  AND  smoothness < 0.12  AND  ZCR > 0.03
    → SINE   (синусоидальный)

ELSE
    → UNKNOWN
```

Порог для каждого признака выбран так, чтобы классы не перекрывались на типовых сигналах проекта.

---

#### Шаг 3 — Выбор оптимального фильтра

| Класс сигнала | Выбранный фильтр | Обоснование |
|---------------|-----------------|-------------|
| **SINE** | `WienerFilter(16, 9, 1e-4)` | Классический Винер оптимален для гауссова шума на гладком стационарном сигнале (MMSE-критерий) |
| **SQUARE** | `MedianFilter(5)` | Медиана сохраняет резкие фронты меандра, не «размывая» переходы |
| **TRIANGLE** | `SavgolFilter(11, 1)` | SavGol poly=1 — кусочно-линейная аппроксимация, точно восстанавливает линейные участки и изломы |
| **ECHO** | `KalmanFilter(0.01, 0.5, 1.0)` | Малый шум процесса → фильтр «следит» за редкими пиками и подавляет шум в паузах |
| **NOISY** | `RobustWienerFilter(10, 5, 1e-4, 3.5, 11)` | MAD-детекция + медианная оценка d[n] + zero-padding: специально спроектирован для несинхронных помех |
| **UNKNOWN** | `MedianFilter(7)` | Безопасный универсальный выбор |

## 🔧 Реализованные алгоритмы

### 1. Медианный фильтр
- **Принцип**: Замена каждого значения медианой соседних значений
- **Преимущества**: Высокая скорость, простота, эффективность против одиночных импульсов
- **Варианты**: Размеры окна 3, 5, 7, 9

### 2. Адаптивный фильтр Винера (`WienerFilter`)
- **Принцип**: Решение уравнения Винера-Хопфа R·w = p, где R — автокорреляционная матрица входного сигнала, p — взаимная корреляция с желаемым сигналом d[n] (скользящее среднее)
- **Преимущества**: MMSE-оптимальность при гауссовом шуме
- **Недостаток**: плохо справляется с несинхронными импульсными помехами

### 3. Робастный фильтр Винера (`RobustWienerFilter`) — **новый**
- **Принцип**: Тот же Винера-Хопф, но с тремя улучшениями для импульсных помех:
  1. **Медианная оценка d[n]** — скользящая медиана вместо среднего (устойчива к выбросам)
  2. **Предварительная MAD-очистка** — `OutlierDetection` удаляет импульсы до построения матрицы R
  3. **Zero-padding** — нулевое дополнение вместо прижимания к `x[0]` на краях
- **Преимущества**: Сохраняет оптимальность Винера для гауссовой части шума, нейтрализует импульсные выбросы

### 3. Морфологические фильтры
- **Принцип**: Операции математической морфологии (эрозия, дилатация, размыкание, замыкание)
- **Преимущества**: Сохранение геометрических свойств, эффективность для помех определенной формы
- **Варианты**: Opening и Closing с различными структурирующими элементами

### 4. Фильтр Савицкого-Голая (`SavgolFilter`)
- **Принцип**: Локальная полиномиальная аппроксимация методом наименьших квадратов
- **Преимущества**: Превосходное сохранение формы сигнала, эффективное сглаживание
- **Варианты**: Различные размеры окна и порядки полинома (9-13 точек, 2-4 степень)

### 5. Алгоритм обнаружения и замещения выбросов (`OutlierDetection`)
- **Принцип**: Двухэтапная обработка - обнаружение помех и их интерполяция
- **Преимущества**: Высокая точность, минимальное искажение полезного сигнала
- **Варианты**: 3 метода обнаружения × 4 метода интерполяции

## 📊 Система оценки

Каждый алгоритм оценивается по четырем ключевым метрикам:

- **SNR (дБ)** - Отношение сигнал/шум после фильтрации
- **MSE** - Среднеквадратичная ошибка восстановления
- **Корреляция** - Сохранение формы исходного сигнала (0-1)
- **Время выполнения (мкс)** - Вычислительная эффективность
